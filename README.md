#  Integration Design
##  Integration Architecture
I’ll use Make.com to build this integration because it’s really effective for chaining multiple tasks in a single automated flow. When a patient submits the consultation form, the data is sent to a Webhook URL, which immediately kicks off the scenario. From there, the data is processed to create or update the contact record in HubSpot, simultaneously triggering the WhatsApp confirmation via the Karix API, and finally pushing a conversion signal back to Google Ads. This centralized approach ensures that all systems are updated in near real-time without needing complex custom middleware.  
##  HubSpot Deduplication Strategy
The biggest technical hurdle here is that HubSpot usually deduplicates contacts based on email addresses. Since we are only capturing phone numbers in this campaign, this will definitely cause duplicate records if users submit multiple times. To fix this, I will add a "Search Contact" API step in my Make scenario that looks for the phone number before creating anything new. This way, I can just update the existing contact record if a match is found, keeping the CRM clean and organized.
##  Monitoring and Fallbacks
To meet the strict 2-minute SLA for WhatsApp, I have to be very careful about API latency. If Karix or Make.com faces a delay, I will use an error handler in Make that notifies the team via email if the process fails or exceeds the time limit. As a secondary fallback, I would configure the system to trigger a standard SMS confirmation if the WhatsApp message fails to deliver, ensuring the patient always receives their details, no matter what.
